# Engineering knowledge

Joka Source projects should compound engineering knowledge instead of solving
the same class of failure independently. This directory contains public,
contributor-safe practices distilled from real products.

## Current references

- [Product development playbook](product-development-playbook.md)
- [Joka CV engineering case study](case-studies/joka-cv.md)
- [Secrets policy](../ops/secrets-policy.md)

## Promotion rule

Promote a practice here only when:

1. it solved a recurring problem in a real project;
2. the source project retains implementation and verification evidence;
3. the generalized rule does not expose credentials, private topology,
   personal data, or private product content;
4. future projects can verify it rather than merely repeat the prose;
5. older conflicting guidance is retired or explicitly superseded.

Project-specific commands and safety boundaries remain beside the project
code. This handbook provides the shared baseline, not a remote replacement for
local ownership.
