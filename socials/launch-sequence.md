## Open Decisions

- **Handle registration**: needs a founder to actually claim `jokasource` / `jokasourcefdn` across platforms — see `socials/MASTER-REGISTRY.md`. Not done as part of this strategy per the "propose, don't register" constraint.
- **Who narrates**: Founder Notes stays first-person (Krishna / Satyam by name, see `roster/founders.yml`). Build Log and Open Invitation run as a faceless "JSF" brand voice so cadence doesn't bottleneck on one person's availability.
- **Joka Connect content placement**: recommend it stays on JSF main channels but framed strictly as "alumni network / professional connection," not dating. The dating angle, if pursued at all, should get its own channel later so it doesn't collide with JSF's sober-OSS-foundation voice (see `projects/joka-connect/PII-POLICY.md` for the underlying privacy constraints). Needs an explicit founding-cell call before post #8 goes out.
