# Social Media Voice Guide

How JSF shows up on social media. See `brand/voice-tone.md` for the foundational brand voice; this doc is the social-specific layer.

## Voice Principles

- **Alumni-rooted, not institution-rooted.** We speak as a few Joka alums building tools, never as "IIM Calcutta official." *Do:* "We built this because we needed it." *Don't:* "IIM Calcutta presents."
- **Sober over hype.** No growth-hacker language, no "10x," no urgency countdowns. *Do:* "Here's what shipped this week." *Don't:* "This will change everything."
- **OSS-first, credit-first.** Every post that references code links the repo and names contributors. *Do:* "PR by @handle, reviewed by @handle." *Don't:* generic "our team."
- **Proud, not preachy.** Campus references (Joka, LBS, Lake) are texture, not a sales pitch. *Do:* one campus-specific detail per post, in passing. *Don't:* a paragraph of nostalgia.
- **Show the boring parts.** Maintenance, decisions, and trade-offs are content, not something to hide. *Do:* "Why we chose X over Y." *Don't:* only announce finished features.
- **No urgency, no guilt.** Cadence is sustainable; we never frame a gap in posting as a failure. *Do:* "Been heads-down on AlumERP, more soon." *Don't:* apologize for not posting.

## Tone by Platform

| Platform | Tone | Cadence |
|----------|------|---------|
| Instagram | Visual, energetic, community-first | 1x/week (feed/carousel) |
| LinkedIn | Professional, mission-driven, thought leadership | 2x/week |
| Twitter/X | Direct, opinionated, quick | 3x/week |
| Discord | Conversational, helpful, casual | Deferred — stand up once community size warrants moderation overhead |

This is deliberately below what a growth-stage product would run. Two founders shipping code cannot sustain daily multi-channel content; a sustainable cadence beats a broken promise.

## Content Pillars

Four recurring themes. Every social post should map to exactly one.

1. **Build Log** (what's shipping) — PR merges, architecture notes, "why we built it this way," roadmap snapshots. Not: unshipped speculation, competitor comparisons.
2. **Founder Notes** (why, not what) — origin-story fragments, decisions and trade-offs, occasional personal reflection from Krishna or Satyam. Not: generic founder-influencer content unrelated to JSF.
3. **Alumni Spotlight** (the network, not the founders) — contributor interviews, alum use-case stories, "who's building this." Not: alumni career content with no JSF tie-in.
4. **Open Invitation** (how to join) — good-first-issue call-outs, "we need help with X," contribution-guide walkthroughs. Not: generic OSS evangelism with no link back to a JSF repo.

## Crisis Response

If a post generates negative backlash or a sensitive situation:
1. Do not delete or edit without founding cell approval
2. Notify the founding cell immediately
3. Do not respond individually — draft a response together
4. Document the incident in `governance/meeting-notes/`
